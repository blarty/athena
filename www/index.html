Athena 1.0 beta 3 released!

Athena is a transactional bridge that allows your databases and other non-RDBMS datasources to participate in Jini distributed transactions.
It is designed to be a universal service front-end with a pluggable host connection to any datasource

Athena 1.0 beta 3 has been released. Get it here
Changes in Beta 3
 
  • Added support for (get)(set)BinaryStream for manipulating binary objects
  • Support for declaring prepared statements
  • Changed handler XML structure to support above
Changes in Beta 2
 
  • Added support for datasource specific actions in shut down hooks after Jini termination
  • Support for Embedded databases
  • Scripts moved from <athenaroot>/bin to <athenaroot>
  • New scripts added for re-running post install and Configuration editor
Changes in Beta 1
 
  • Support for secure exporters for Connections instances
  • Fixed memory leak for exporters such as transaction participants being held in memory too long - not being explicitly unexported
  • Ability to schedule exported proxies to be unexported and cleared up after a set time
  • Modified startup to use Configurations more
  • A proper installer - just double click or execute the jar.
  • More configurable logging and System.out=>Logging changes.
  • Better offline checking - when your data source goes offline, Athena leaves the groups it is registered with, rejoining when the datasource is available again.
